## Executive Summary

This contract defines the production hardening work for rust-sitter v1.0.0, establishing long-term stability guarantees and GLR-specific capabilities that go beyond Tree-sitter.

**Goal**: Ship a **production-ready 1.0 release** with:
- Stable API (semver enforcement)
- Security audit passed
- ≥3 production deployments
- ≥10 community grammars
- GLR-specific features (parse forests, disambiguation, error recovery)

**Timeline**: 12 weeks (Q4 2026)

## I. Acceptance Criteria

### AC-PROD1: API Stability & Semver Guarantees

**Requirement**: Freeze public API and establish semver policy

**Semver Policy**:
- **Breaking changes**: Only in major versions (2.0, 3.0, etc.)
- **New features**: Minor versions (1.1, 1.2, etc.)
- **Bug fixes**: Patch versions (1.0.1, 1.0.2, etc.)
- **Deprecation**: 3 months minimum notice

**API Freeze**:
- Public API review (external contributors)
- API documentation: 100% coverage
- Stability guarantees document

**Success Criteria**:
- [ ] API frozen (no breaking changes post-1.0.0)
- [ ] Semver policy documented
- [ ] Deprecation policy established
- [ ] API stability guarantees published

### AC-PROD2: Parse Forest API (GLR-Specific)

**Requirement**: Expose parse forests for ambiguous grammars, going beyond Tree-sitter

**API Design**:
```rust
// runtime2/src/forest.rs
pub struct ForestHandle {
    forest: ParseForest,
}

impl ForestHandle {
    /// Get number of ambiguous interpretations
    pub fn ambiguity_count(&self) -> usize;

    /// Enumerate all root alternatives
    pub fn root_alternatives(&self) -> Vec<Node>;

    /// Export forest as Graphviz DOT
    pub fn to_graphviz(&self) -> String;

    /// Resolve to a single tree using disambiguation strategy
    pub fn resolve(&self, strategy: DisambiguationStrategy) -> Tree;

    /// Get ambiguity diagnostics
    pub fn ambiguity_report(&self) -> AmbiguityReport;
}

pub enum DisambiguationStrategy {
    PreferShift,        // Favor shift over reduce
    PreferReduce,       // Favor reduce over shift
    LongestMatch,       // Choose longest derivation
    Custom(Box<dyn Fn(&[Tree]) -> Tree>), // User-defined
}

pub struct AmbiguityReport {
    pub total_ambiguities: usize,
    pub ambiguous_regions: Vec<Range>,
    pub conflicting_rules: Vec<(RuleId, RuleId)>,
}
```

**Success Criteria**:
- [ ] Forest API implemented
- [ ] Ambiguity diagnostics functional
- [ ] Graphviz export working
- [ ] Disambiguation strategies tested

### AC-PROD3: Advanced Error Recovery

**Requirement**: Robust, configurable error recovery strategies

**Error Recovery Strategies**:
```rust
pub enum RecoveryStrategy {
    /// Skip tokens until synchronization point
    SkipToSync,

    /// Insert missing tokens
    InsertMissing,

    /// Use error recovery rules from grammar
    GrammarRules,

    /// Maintain structural integrity
    StructuralGuarantees {
        keep_outer_structure: bool,
        max_skip_distance: usize,
    },
}

impl Parser {
    pub fn set_recovery_strategy(&mut self, strategy: RecoveryStrategy);
    pub fn get_error_report(&self) -> ErrorReport;
}

pub struct ErrorReport {
    pub errors: Vec<ParseError>,
    pub recovery_actions: Vec<RecoveryAction>,
    pub structural_integrity: StructuralIntegrityReport,
}
```

**Success Criteria**:
- [ ] Multiple recovery strategies implemented
- [ ] Configurable per-parser
- [ ] Error reports with actionable information
- [ ] Corpus of broken code examples with expected behavior

### AC-PROD4: Multi-Language Composition

**Requirement**: Support embedded languages and grammar composition

**Composition API**:
```rust
// ir/src/composition.rs
pub struct ComposedGrammar {
    pub host: Grammar,
    pub injections: Vec<Injection>,
}

pub struct Injection {
    pub language: Grammar,
    pub injection_points: Vec<InjectionPoint>,
    pub conflict_resolution: ConflictResolution,
}

pub enum InjectionPoint {
    NodeType(String),    // e.g., "script_tag" in HTML
    Field(String),       // e.g., field "body" in nodes
    Custom(Box<dyn Fn(&Node) -> bool>),
}

pub enum ConflictResolution {
    PreferHost,
    PreferInjected,
    GLRFork,  // Let GLR handle ambiguity
}
```

**Examples**:
- HTML with embedded JS/CSS
- Markdown with code blocks
- Template languages (Jinja, ERB)

**Success Criteria**:
- [ ] Grammar composition working
- [ ] Example: HTML + JS + CSS
- [ ] GLR handling of overlapping syntaxes
- [ ] Documentation with examples

### AC-PROD5: Observability & Production Controls

**Requirement**: Metrics, tracing, timeouts, and safe fallbacks for production use

**Observability**:
```rust
// runtime2/src/metrics.rs
pub struct ParserMetrics {
    pub parse_count: u64,
    pub total_time_us: u64,
    pub avg_time_us: f64,
    pub peak_memory_bytes: u64,
    pub incremental_reuse_avg: f64,
    pub error_count: u64,
}

pub trait MetricsCollector {
    fn record_parse(&mut self, duration_us: u64, reuse_percent: f64);
    fn record_error(&mut self, error: &ParseError);
}

impl Parser {
    pub fn set_metrics_collector(&mut self, collector: Box<dyn MetricsCollector>);
    pub fn get_metrics(&self) -> &ParserMetrics;
}
```

**Production Controls**:
```rust
impl Parser {
    /// Set maximum parse time (timeout)
    pub fn set_timeout(&mut self, timeout_ms: u64);

    /// Set maximum memory usage
    pub fn set_memory_limit(&mut self, limit_bytes: u64);

    /// Set step limit (for adversarial inputs)
    pub fn set_step_limit(&mut self, steps: u64);

    /// Enable best-effort mode (partial parse on timeout)
    pub fn set_best_effort(&mut self, enabled: bool);
}
```

**Success Criteria**:
- [ ] Metrics collection functional
- [ ] Timeouts and limits enforced
- [ ] Best-effort mode returns partial results
- [ ] Production deployment guide

**Requirement**: Pass security audit and address findings

**Security Checklist**:

## VI. Risk Assessment

### High Risks

**1. API Instability**
- **Risk**: Breaking changes required post-1.0
- **Mitigation**: Extensive review, deprecation period, compatibility shims
- **Fallback**: Mark as 1.0-beta, extend review period

**2. Security Findings**
- **Risk**: Critical vulnerabilities found in audit
- **Probability**: Medium
- **Impact**: Critical (blocks 1.0 release)
- **Mitigation**: Internal review first, allocate time for fixes
- **Fallback**: Delay 1.0 until all critical findings addressed

### Medium Risks

**1. Production Adoption**
- **Risk**: Struggle to find 3 production deployments
- **Mitigation**: Dogfood internally, reach out to early adopters
- **Fallback**: Reduce to 2 case studies if necessary

**2. Community Grammars**
- **Risk**: <10 community grammars
- **Mitigation**: Outreach, contribution sprints, good docs
- **Fallback**: Official team creates more grammars initially
