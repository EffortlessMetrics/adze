### AC-PERF4: GLR Stack Pooling

**Requirement**: Eliminate allocation overhead in GLR fork/merge operations

**Design**:
```rust
// glr-core/src/stack_pool.rs
pub struct StackPool {
    free_stacks: Vec<Box<ParserStack>>,
    max_pool_size: usize,
}

impl StackPool {
    pub fn acquire(&mut self) -> Box<ParserStack> {
        self.free_stacks.pop().unwrap_or_else(|| {
            Box::new(ParserStack::new())
        })
    }

    pub fn release(&mut self, mut stack: Box<ParserStack>) {
        if self.free_stacks.len() < self.max_pool_size {
            stack.clear();
            self.free_stacks.push(stack);
        }
    }
}
```

**Integration Points**:
- `glr-core/src/lib.rs`: Use pool for fork operations
- `runtime2/src/engine.rs`: Initialize pool at parser creation
- Benchmark: Compare pooled vs unpooled on ambiguous grammars

**Performance Target**:
- ≥40% reduction in allocation count during GLR parsing
- ≥15% speedup on fork-heavy grammars (dangling-else)
- No performance regression on non-forking grammars

**Success Criteria**:
- [ ] StackPool implemented and tested
- [ ] Benchmark comparison: pooled vs unpooled
- [ ] Performance targets met (40% alloc, 15% speed)
- [ ] Thread-safe for future parallelization
- [ ] Memory bounded (max_pool_size prevents unbounded growth)

## III. Fixtures & Benchmark Targets

### Fixture Requirements

**Python** (tree-sitter-python grammar):
- **Small** (100 LOC): Fast iteration, baseline (~5-10µs target)
- **Medium** (2000 LOC): Real-world scale (~100-200µs target)
- **Large** (10000 LOC): Stress test (~500-1000µs target)

**JavaScript** (tree-sitter-javascript grammar):
- **Medium** (1000 LOC): Real-world scale (~50-100µs target)
- **Large** (5000 LOC): Stress test (~250-500µs target)

**Rust** (optional, future):
- **Medium** (2000 LOC): Complex grammar test

### Fixture Sources

1. **Extract from open-source projects** (with attribution):
   - Python: Django, Flask, pytest codebase snippets
   - JavaScript: React, Vue, Node.js snippets

2. **Synthetic fixtures** (for controllable LOC):
   - Generate Python classes, functions, control flow
   - Validate syntax with `python -m py_compile`

3. **Fixture validation**:
   ```rust
   // benchmarks/fixtures/mod.rs
   pub struct Fixture {
       pub name: &'static str,
       pub language: Language,
       pub content: &'static str,
       pub expected_loc: usize,
       pub license: &'static str, // e.g., "MIT", "Apache-2.0"
   }

   #[test]
   fn validate_fixture_loc() {
       for fixture in ALL_FIXTURES {
           let actual_loc = fixture.content.lines().count();
           assert!(
               (actual_loc as isize - fixture.expected_loc as isize).abs() < 10,
               "{} LOC mismatch: expected {}, got {}",
               fixture.name, fixture.expected_loc, actual_loc
           );
       }
   }
   ```

**Success Criteria**:
- [ ] All fixtures have licenses and attribution
- [ ] LOC counts validated (within 10 lines of expected)
- [ ] Files parse successfully with tree-sitter-c (ground truth)
- [ ] Files parse successfully with adze

## VIII. Risk Assessment

### High Risks

**1. Performance Targets Missed**
- **Risk**: Cannot achieve ≤2× Tree-sitter C
- **Probability**: Medium
- **Mitigation**: Incremental optimization, frequent benchmarking
- **Fallback**: Document achieved performance, adjust targets for v0.9.0

**2. Arena/Pool Complexity**
- **Risk**: Lifetime issues or memory unsafety
- **Probability**: Medium
- **Mitigation**: Extensive testing, Miri validation, careful design reviews
- **Fallback**: Simpler optimization strategies (e.g., Vec pooling only)

### Medium Risks

**1. Fixture Licensing**
- **Risk**: Cannot find suitable open-source fixtures
- **Probability**: Low
- **Mitigation**: Generate synthetic fixtures, contact authors for permission
- **Fallback**: Use smaller fixtures or proprietary ones for internal testing only

**2. CI Performance Variability**
- **Risk**: Noisy benchmarks cause false positives/negatives
- **Probability**: Medium
- **Mitigation**: Multiple samples, statistical analysis, machine-specific baselines
- **Fallback**: Manual benchmark review for CI failures
