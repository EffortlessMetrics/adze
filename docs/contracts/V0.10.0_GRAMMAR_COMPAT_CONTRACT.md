## Executive Summary

This contract defines the Tree-sitter grammar compatibility work for rust-sitter v0.10.0, enabling rust-sitter to consume existing Tree-sitter grammars and produce identical parse trees.

**Goal**: Achieve **≥95% test compatibility** with tree-sitter-python test suite and provide a clear migration path for grammar authors.

**Timeline**: 8 weeks (Q2 2026)

---

## I. Acceptance Criteria

### AC-GRAM1: Grammar IR (Neutral Representation)

**Requirement**: Define a neutral intermediate representation that can model both Tree-sitter and rust-sitter grammar features

**IR Design**:
```rust
// ir/src/grammar_ir.rs
pub struct GrammarIR {
    pub name: String,
    pub rules: Vec<Rule>,
    pub externals: Vec<ExternalRule>,
    pub extras: Vec<Symbol>,
    pub conflicts: Vec<ConflictSet>,
    pub precedences: Vec<PrecedenceDecl>,
    pub word: Option<Symbol>,
    pub supertypes: Vec<Symbol>,
}

pub enum Rule {
    Symbol(Symbol),
    String(String),
    Pattern(Regex),
    Sequence(Vec<Rule>),
    Choice(Vec<Rule>),
    Repeat(Box<Rule>),
    Repeat1(Box<Rule>),
    Optional(Box<Rule>),
    Alias(Box<Rule>, String),
    Field(String, Box<Rule>),
    Prec(i32, Box<Rule>),
    PrecLeft(i32, Box<Rule>),
    PrecRight(i32, Box<Rule>),
}
```

**Success Criteria**:
- [ ] IR can represent all TS grammar constructs
- [ ] IR can represent rust-sitter extensions (GLR-specific features)
- [ ] IR is serializable (JSON/TOML)
- [ ] IR validation ensures well-formed grammars

---

### AC-GRAM2: TS → IR Converter

**Requirement**: Convert Tree-sitter `grammar.js` files to Grammar IR

**Implementation**:
```bash
cargo xtask ts-import-grammar <repo-url>
# → Clones repo
# → Parses grammar.js
# → Generates .rsir file
# → Optionally generates rust-sitter grammar code
```

**Conversion Process**:
1. Parse `grammar.js` (JavaScript AST)
2. Extract grammar object
3. Convert TS constructs to IR constructs
4. Validate IR
5. Output `.rsir` file

**Success Criteria**:
- [ ] Can convert tree-sitter-python grammar
- [ ] Can convert tree-sitter-javascript grammar
- [ ] Conversion preserves all semantic information
- [ ] Validation catches malformed grammars

---

### AC-GRAM3: IR → rust-sitter Compiler

**Requirement**: Generate rust-sitter parser from Grammar IR

**Compilation Pipeline**:
```
.rsir → IR Validation → Conflict Analysis → Table Generation → Code Generation → .rs
```

**Output**:
- Rust source file with grammar types
- Parse tables (`.parsetable` format)
- Node types metadata (`.json`)

**Success Criteria**:
- [ ] Compiles Python grammar successfully
- [ ] Generated parser produces same node types as TS
- [ ] Field names match TS field names
- [ ] Precedence/associativity preserved

### AC-GRAM4: TS Runtime Compatibility API

**Requirement**: Provide Tree-sitter-compatible runtime API

**Implementation** (`runtime/src/ts_compat.rs`):
```rust
pub mod ts {
    pub struct Parser { /* ... */ }
    pub struct Language { /* ... */ }
    pub struct Tree { /* ... */ }
    pub struct Node<'tree> { /* ... */ }
    pub struct TreeCursor<'tree> { /* ... */ }

    impl Parser {
        pub fn new() -> Self;
        pub fn set_language(&mut self, language: Language) -> Result<(), LanguageError>;
        pub fn parse(&mut self, input: &[u8], old_tree: Option<&Tree>) -> Option<Tree>;
    }

    impl Tree {
        pub fn root_node(&self) -> Node<'_>;
        pub fn edit(&mut self, edit: &InputEdit);
    }

    impl Node<'_> {
        pub fn kind(&self) -> &str;
        pub fn start_byte(&self) -> usize;
        pub fn end_byte(&self) -> usize;
        pub fn child_count(&self) -> usize;
        pub fn child(&self, index: usize) -> Option<Node<'_>>;
        pub fn field_name_for_child(&self, index: usize) -> Option<&str>;
    }
}
```

**Success Criteria**:
- [ ] API matches Tree-sitter C API semantics
- [ ] Rust API is idiomatic and safe
- [ ] Performance within 2× of Tree-sitter C
