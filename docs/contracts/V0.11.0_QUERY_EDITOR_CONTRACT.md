## I. Acceptance Criteria

### AC-QUERY1: Query Engine (S-expression Parser)

**Requirement**: Implement Tree-sitter query language parser and evaluator

**Query Language** (`.scm` files):
```scheme
; Capture function definitions
(function_definition
  name: (identifier) @function.name
  parameters: (parameters) @function.params
  body: (block) @function.body)

; Match with predicates
((identifier) @constant
 (#match? @constant "^[A-Z_]+$"))

; Multiple captures
(call_expression
  function: (identifier) @function
  arguments: (argument_list) @args)
```

**Implementation**:
```rust
// runtime/src/query.rs
pub struct Query {
    patterns: Vec<Pattern>,
    captures: Vec<Capture>,
}

impl Query {
    pub fn new(language: &Language, source: &str) -> Result<Self, QueryError>;
    pub fn matches<'tree>(&self, node: Node<'tree>) -> impl Iterator<Item = Match<'tree>>;
    pub fn captures<'tree>(&self, node: Node<'tree>) -> impl Iterator<Item = Capture<'tree>>;
}

pub struct Match<'tree> {
    pub pattern_index: usize,
    pub captures: Vec<Capture<'tree>>,
}

pub struct Capture<'tree> {
    pub name: String,
    pub node: Node<'tree>,
}
```

**Success Criteria**:
- [ ] S-expression parser implemented
- [ ] Pattern matching algorithm functional
- [ ] Predicates: `#match?`, `#eq?`, `#any-of?`, `#is?`, `#is-not?`
- [ ] Performance: ≤2× Tree-sitter C on typical queries

### AC-QUERY3: LSP/Daemon Implementation

**Requirement**: Implement language server protocol support using adze

**LSP Features**:
```rust
// lsp/src/server.rs
pub struct AdzeLSP {
    parser: Parser,
    trees: HashMap<Url, Tree>,
    queries: HashMap<String, Query>, // "highlighting", "folding", etc.
}

impl AdzeLSP {
    // Document lifecycle
    pub fn did_open(&mut self, params: DidOpenTextDocumentParams);
    pub fn did_change(&mut self, params: DidChangeTextDocumentParams);
    pub fn did_close(&mut self, params: DidCloseTextDocumentParams);

    // Language features
    pub fn semantic_tokens(&self, uri: &Url) -> Vec<SemanticToken>;
    pub fn folding_ranges(&self, uri: &Url) -> Vec<FoldingRange>;
    pub fn document_symbols(&self, uri: &Url) -> Vec<DocumentSymbol>;
}
```

**Success Criteria**:
- [ ] LSP server binary (`adze-lsp`)
- [ ] Supports: syntax highlighting, folding, outline
- [ ] Incremental parsing for document changes
- [ ] Response time: <50ms for typical operations

**Requirement**: Provide tools to ease migration from Tree-sitter to adze

**Tools**:
```bash

# Compare parsing results

# → Parses with both TS and adze
# → Compares parse trees
# → Reports differences

# Validate query compatibility
cargo xtask ts-query-check queries/highlights.scm
# → Checks query against adze implementation
# → Reports unsupported features
```

**Success Criteria**:
