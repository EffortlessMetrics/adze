# adze-runtime

GLR runtime for adze parsers with Tree-sitter API compatibility.

## Overview

This crate provides a runtime that mimics Tree-sitter's API while using GLR (Generalized LR) parsing internally to handle ambiguous grammars. It's designed to be a drop-in replacement for Tree-sitter's runtime API.

## Features

- **Tree-sitter compatible API**: `Parser`, `Tree`, `Node`, `Language` types
- **GLR parsing**: Handles ambiguous grammars with multiple parse paths
- **Incremental parsing**: Efficient re-parsing of edited text (feature-gated)
- **External scanners**: Support for custom lexing logic
- **Arena allocation**: Optional performance optimizations

## Quick Start

```rust
use adze_runtime::{Parser, Language};

// Create a parser
let mut parser = Parser::new();

// Set the language (would come from a generated grammar crate)
let language = your_grammar::language();
parser.set_language(language)?;

// Parse some text
let tree = parser.parse_utf8("def hello(): pass", None)?;

// Walk the syntax tree
let root = tree.root_node();
println!("Root kind: {}", root.kind());

for i in 0..root.child_count() {
    if let Some(child) = root.child(i) {
        println!("Child {}: {}", i, child.kind());
    }
}
```

## Architecture

The runtime is structured as follows:

- **Parser**: Main parsing interface, manages language and parsing state
- **Language**: Contains parse tables and grammar metadata
- **Tree**: Represents a parsed syntax tree (internally may be an SPPF for ambiguous parses)
- **Node**: A node in the syntax tree with Tree-sitter-compatible traversal API
- **ExternalScanner**: Trait for custom lexing logic (e.g., indentation)

## Integration with adze

This runtime is designed to work with parsers generated by adze's build tools. A typical workflow would be:

1. Define your grammar using adze macros
2. Generate parser tables at build time
3. Link with this runtime
4. Use the Tree-sitter-compatible API to parse text
